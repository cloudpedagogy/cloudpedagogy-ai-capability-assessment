# AI Capability Assessment — User Instructions

---
### 2. What This Tool Does
This application is a frontline reflective tool used by academic teams or departments to assess their practical, day-to-day capability with AI. It measures readiness across six core domains, establishing a foundational baseline for capability development.

---
### 3. Role in the Ecosystem
- **Phase:** Phase 3 — Capability System
- **Role:** Reflective assessment of organizational AI capability domains.
- **Reference:** [../SYSTEM_OVERVIEW.md](../SYSTEM_OVERVIEW.md)

---
### 4. When to Use This Tool
- When a new team or department needs to baseline their current AI literacy.
- As a prerequisite exercise before deploying new AI tools to a specific faculty.
- To gather grassroots data on how AI is actually being perceived and used on the ground.

---
### 5. Inputs
- Manual survey completion by individual faculty members or teams.

---
### 6. How to Use (Step-by-Step)
1. Distribute the assessment to the target group.
2. Users complete the prompts regarding Awareness, Co-agency, Practice, Ethics, Governance, and Renewal.
3. Users answer honestly, capturing their current confidence and practical constraints.
4. Users submit the assessment to generate a localized capability score.
5. Export the resulting JSON file for institutional aggregation.

---
### 7. Key Outputs
- Individual or localized team capability scores.
- A standardized JSON payload representing that specific unit's readiness.

---
### 8. How It Connects to Other Tools
- **Upstream:** N/A
- **Downstream:** Outputs from this tool must be imported into the **Capability Dashboard** and **Gaps & Risk** tools for analysis.

---
### 9. Limitations
- Highly localized; a single assessment does not represent the entire institution.
- It is a reflective starting point, not an objective technical test.

---
### 10. Tips
- Ensure users understand this is a *developmental diagnostic*, not a performance evaluation, to encourage honesty.
